# 論文紹介"Prioritized memory access explains planning and hippocampal replay"

## 文献情報

### 著者: Mattar, Marcelo G., and Nathaniel D. Daw.

### 雑誌名: Nature neuroscience 

### 巻: 21

### 記事番号: 11

### 出版年: 2018


## イントロ

### 背景
生き物がうまく行動するためには、過去の経験をもとに、将来より多くの報酬が得られる行動を選ぶ必要がある。
しかし、迷路探索課題のように、行動のたびに報酬が与えられず、最後の結果を見て初めて良し悪しが決まる課題では、その場では各行動の価値が分かりにくい。
そのため、このような課題では、過去の経験をつなげて先の結果を予測しながら行動を選ぶ必要がある。

### 計画と事前評価
また、経験の利用は行動の直前に限られず、より前の段階から将来の行動評価や意思決定の準備に用いられている可能性がある。
このとき、脳は蓄積されたすべての経験を同時に参照できるわけではないため、どの経験を優先して参照するかが、その後の行動選択に影響すると考えられる。
したがって、より大きな報酬をもたらす選択を実現するうえで、脳が各時点でどの経験を参照するかが重要になる。

### 海馬と場所細胞
この仕組みを考えるうえで重要なのが海馬である。
海馬には、動物が特定の場所にいるときに強く活動する場所細胞が存在する。
動物が移動すると、強く活動する細胞も場所に応じて切り替わるため、海馬は細胞活動の組み合わせやその順序によって位置情報を表していると考えられる。
たとえば、地点 A・B・C に対応する場所細胞があるとすると、動物の移動に合わせて、強く活動する細胞も A、B、C…と順に切り替わっていく。

### リプレイ
海馬では、動物が立ち止まっているときや休息中にも、場所細胞の発火の並びが短時間で再生されることがあり、これをリプレイという。
リプレイには、進行方向に沿って再生される順方向リプレイ、通ってきた経路を逆向きにたどる逆方向リプレイ、睡眠や休息中に起こるオフラインリプレイなどがある。これらは計画、学習などと関係すると考えられてきたが、どの記憶をどの順序で再生すればよいかを一つの枠組みで説明する理論は十分ではなかった。

### 本論文の提案
この論文は、強化学習に基づき、将来の報酬を最大化するために各時点でどの記憶を想起すべきかを定式化した。各経験の有用性は、予想外の結果が起きたとき、その結果につながった少し前の状態や行動を優先して更新する gain と、今後必要になる状態を重視する need の積で表される。これによりリプレイを、単なる計画ではなく、経験から価値を更新し、その後の意思決定を支える仕組みとして説明される。

## 手法

### 環境
#### グリッドワールド(図1-c)
- 障害物を含むグリッドワールド環境を用いる。
- 行動は上下左右の4方向とする。
- 各状態における行動価値関数を softmax 法によって行動選択確率に変換し、その確率に基づいて行動を決定する。
- 開始位置は各エピソードでランダムに設定する。
- 報酬には、平均 1、標準偏差 0.1 のガウスノイズを加える。

#### 直線路(図1-c)
- 右向きコースと左向きコースを用意し、ゴールに到達するとコースが入れ替わるループ課題を構成する。
- 同じ位置であっても、「右向きに進んでいるとき」と「左向きに戻っているとき」を区別する。
- これは、状態空間が位置と進行方向の両方を区別できるようにするためであり、forward replay と reverse replay を明確に区別するためでもある。

### backup / replay の要点
- エージェントは、実際に行動して学習する過程と、記憶内の経験を replay して学習する過程の両方をもつ。
- replay は各エピソードの開始時と終了時に、それぞれ 20 回ずつ行う。
- 各 replay においては、記憶内にあるすべての 1-step 経験が候補となる。これは現在の状態だけでなく、全状態・全行動に対応する経験全体を指す。
- その中から Gain × Need が最大となる経験を選び、Q 値の更新に用いる。
- ここでいう backup とは、実際に行動することではなく、記憶内の経験を用いて Q 値を更新することを意味する。

### 1-step backup
- 単独の経験を用いる場合には、その経験に基づいて 1 回分の価値更新を行う。

### n-step backup
- 直前に backup した経験列の最適な続きとなる経験がある場合には、backup を 1 ステップ先ではなく系列として延長できる。
- この場合、系列の先頭だけでなく、途中に含まれるすべての行動価値関数もまとめて更新する。
- ただし、最初の行動の後に続く行動列は、現在の最適方策に沿っている必要がある。
- 系列を 1 ステップ先に伸ばす候補については、Gain 項として系列全体で生じる改善量の総和をとり、Need 項として追加した先端状態の将来訪問見込み回数を用いて EVB を計算し、その経験を付け加えるかどうかを決定する。
- EVB が同じ場合には、より短い系列を優先する。
- 有意な replay は、長さ 5 以上の系列として扱う。(図２a,g)

### 初期化
- エージェントは初期状態から、全状態・全行動に対応する経験集合をもっている。
- この経験集合には、「どこに遷移するか」という情報だけでなく、「どのような報酬が得られるか」という情報も含まれる。
- ただし、経験を最初からすべて保持していても、それらをどの順序で何回用いるかは planning step によって決まる。
